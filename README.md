# найди-виралку - парсер виралок (find-only)

Скилл для Claude Code: находит чужие ВИРАЛЬНЫЕ материалы по твоей теме и отдаёт ссылки. Сам ничего не скачивает и не переделывает - только ищет и даёт список ссылок, чтобы ты увидел, что уже залетело в твоей нише, и взял за референс.

## Что именно ищет

Ты сам выбираешь ФОРМАТ - скилл ищет то, что попросишь:
- видео-ролики: Reels, YouTube Shorts, обычные YouTube-видео
- карусели (Instagram)

Плюс тему и (по желанию) критерии. Скилл вернёт ссылки на самые виральные единицы по этим параметрам.

Честно про охват (скилл работает без логина, на публичном поиске):
- видео / Reels / Shorts / YouTube - находит надёжно
- карусели (Instagram) и TikTok - находит через веб-поиск публичные примеры, анонимно не так полно, как видео (для сплошного обхода каруселей нужен залогиненный инструмент)

## Установка

Нужен установленный [Claude Code](https://docs.claude.com/claude-code). Затем:

```bash
cd ~/.claude/skills && git clone https://github.com/theivansergeev/naydi-viralku найди-виралку
```

Перезапусти Claude Code - скилл подхватится сам.

Для поиска видео по YouTube нужен `yt-dlp` (без него остаётся веб-поиск):
```bash
brew install yt-dlp     # macOS
pip install -U yt-dlp   # Windows / Linux
```

## Как пользоваться

Скажи Claude ЧТО искать, в каком ФОРМАТЕ и по какой ТЕМЕ. Примеры:

Рилс / Shorts / видео:
- «найди виральные рилс про <твоя тема>»
- «найди топ YouTube Shorts про <тема> за последние 30 дней»
- «найди виральные видео про <тема> с более чем 500 000 просмотров»

Карусели:
- «найди виральные карусели про <тема> в Instagram»

Можно добавить критерии: площадку, минимум просмотров или лайков, свежесть (за сколько дней). Не задашь - возьмутся разумные значения по умолчанию.

## Что получишь

Таблицу ссылок прямо в чат: ссылка | площадка | метрика (просмотры/лайки) | дата | формат. Отсортировано по виральности. Ничего не скачивается и не переделывается - только ссылки.

## Чего скилл НЕ делает

Не скачивает, не транскрибирует, не разбирает на структуру, не уникализирует, не переделывает. Это find-only: нашёл и дал ссылку. Дальше с найденным работаешь сам.
